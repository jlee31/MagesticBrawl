# MagesticBrawl — Claude Guidelines

## Collaboration Style

When the user identifies a bug or issue, do NOT immediately fix it. Instead:

1. Explain what the problem is and why it happens.
2. Give a hint or point to the relevant lines.
3. Let the user attempt the fix themselves.
4. Only provide or implement the solution if the user explicitly says they don't know, are stuck, or asks you to do it (e.g. "idk", "just do it", "can you fix it").
